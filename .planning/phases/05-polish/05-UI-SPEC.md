## Interaction Contracts

Phase 5 adds no new user-facing interaction patterns. The following section **audits and enforces** existing contracts.

### Focus Ring Standard (SEO-02)

All interactive elements must display a visible focus ring on keyboard navigation. This is the canonical standard:

| Element | Expected Focus Style | Implementation |
|---------|---------------------|----------------|
| `NavLink` (`<a>`) | `focus-visible:ring-2 focus-visible:ring-indigo-500 focus-visible:ring-offset-2` | Already implemented in `NavLink.tsx` — verify not overridden |
| `CTAButton` (shadcn `Button` variant="default") | `focus-visible:border-ring focus-visible:ring-3 focus-visible:ring-ring/50` | Built into `button.tsx` CVA base — verify `--ring: #6366f1` resolves correctly |
| `ThemeToggle` (shadcn `Button` variant="ghost" size="icon") | Same as CTAButton — shadcn base CVA | Verify no `outline-none` suppression without replacement |
| `ContactIconButton` (shadcn `Button` variant="outline" size="icon") | Same as CTAButton — shadcn base CVA | Verify wrapper `<a>` does not capture focus instead of inner `Button` |
| `MobileNav` hamburger (shadcn `Button` variant="ghost" size="icon") | Same as CTAButton — shadcn base CVA | Verify |
| `HeroPhoto` (non-interactive) | No focus ring needed — not a focusable element | OK — `<div>` with no tabIndex |

**Visual appearance of focus ring:** 2px (or 3px via shadcn) solid `#6366f1` ring with 2px transparent offset gap, visible in both light and dark modes. Ring must be visible against both `#fafafa` (light background) and `#09090b` (dark background) — `#6366f1` on `#09090b` passes AA at 4.7:1.

**Audit check for `ContactIconButton`:** The outer `<a>` tag is focusable by default. The inner `Button` will also receive focus. This creates a double-focus stop. Resolution: add `tabIndex={-1}` to the inner `Button` so only the `<a>` is in tab order, then apply `focus-visible:ring-2 focus-visible:ring-indigo-500 focus-visible:ring-offset-2 rounded-full` to the `<a>` tag itself.

## Lighthouse CI Contract (D-12, D-13)

Not a visual contract — included here as the performance thresholds that gate deployment.

### Thresholds

| Metric | Threshold | Lighthouse assertion |
|--------|-----------|---------------------|
| LCP | < 2500ms | `maxNumericValue: 2500` |
| CLS | < 0.1 | `maxNumericValue: 0.1` |
| Performance score | >= 0.85 | `minScore: 0.85` |
| Accessibility score | >= 0.9 | `minScore: 0.9` |

### Storage method

`--storage.storageMethod=temporary` — no external LHCI server for v1.0 (D-13).

Lighthouse CI runs as a separate job in `.github/workflows/ci.yml` after the `build` job completes. It uses `yarn build && yarn start` in server mode on port 3000.
