    Step 1 — Create lighthouserc.json in the project root (/home/leforjer/afolarte-portfolio/lighthouserc.json). This is a new file.

    Content: a JSON object with a single "ci" key containing three sub-keys:

    "collect": `{ "startServerCommand": "yarn start", "startServerReadyPattern": "Ready on", "url": ["http://localhost:3000"], "numberOfRuns": 3 }`
    - startServerCommand MUST be "yarn start" (production build), NOT "yarn dev". Running against dev produces artificially low scores (50-60 range) due to unminified builds and HMR overhead.
    - startServerReadyPattern "Ready on" matches Next.js production server startup message.
    - 3 runs for averaging to reduce measurement noise.

    "assert": `{ "assertions": { "largest-contentful-paint": ["error", { "maxNumericValue": 2500 }], "cumulative-layout-shift": ["error", { "maxNumericValue": 0.1 }], "categories:performance": ["error", { "minScore": 0.85 }], "categories:accessibility": ["error", { "minScore": 0.9 }] } }`
    - "error" severity means CI fails if threshold is exceeded.
    - LCP threshold: 2500ms (< 2.5s per LAYOUT-05 requirement)
    - CLS threshold: 0.1 (< 0.1 per LAYOUT-05 requirement)
    - Performance score: 0.85 (gives some headroom while still blocking regressions)
    - Accessibility score: 0.9 (portfolio should score high after Phase 5 a11y fixes)

    "upload": `{ "target": "temporary-public-storage" }`
    - temporary-public-storage is the Vercel-hosted free LHCI storage. No authentication required for basic usage. No LHCI_GITHUB_APP_TOKEN secret needed.

    Step 2 — Append two new steps to .github/workflows/ci.yml, after the "E2E tests" step (currently the last step, line 44-45). Match the existing YAML indentation exactly (6 spaces for step name/run lines — they are under `steps:` which is under the job).

    New steps to append:
    ```
          - name: Install Lighthouse CI
            run: yarn global add @lhci/cli@0.15.1

          - name: Run Lighthouse CI
            run: lhci autorun
    ```

    lhci autorun will:
    1. Start the server using `yarn start` (from lighthouserc.json collect.startServerCommand)
    2. Wait for "Ready on" in stdout (startServerReadyPattern)
    3. Run 3 Lighthouse audits against http://localhost:3000
    4. Assert against the thresholds in lighthouserc.json
    5. Upload results to temporary-public-storage

    Important: `lhci autorun` picks up lighthouserc.json automatically from the project root (same directory where `lhci autorun` is invoked, which is the repo root in GitHub Actions).

    Note: @lhci/cli is added as `yarn global add` in the CI step rather than as a devDependency in package.json. This is acceptable for v1.0 (avoids adding a heavy devDep to local installs). The version is pinned to 0.15.1 for reproducibility.

    No LHCI_GITHUB_APP_TOKEN secret is configured or needed — the `env:` block for the LHCI step is omitted. temporary-public-storage works without authentication.
